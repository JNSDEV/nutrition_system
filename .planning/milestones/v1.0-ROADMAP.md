## Overview

Turn 15 root `.txt` knowledge files into a fully operational markdown-first nutrition system for two people (Jonas + Farva). Markdown-only — no code, no build, no CI. Claude reads/writes files on the laptop; Claude mobile chat acts as a buffer for on-the-go logging.

**Core value:** Reduce decision fatigue and keep both people consistently on plan, week after week, by guiding cook → eat → log → adjust without each step requiring fresh thought.

## Milestone Summary

**Decimal Phases:**
- Phase 4.1: Library Wiring Fixes (inserted after Phase 4 to close audit-discovered BLOCKERs)

**Key Decisions:**
- Markdown-only system — no code, no build, no CI; Claude reads/writes files on laptop
- Two separate per-person trackers, one shared library — different targets, different cycling load
- External app (MFP/Cronometer) owns kcal/macro actuals; library formulas provide estimates; system records both, never recomputes external numbers
- Conversational args for all slash commands (no positional arguments — phone-friendly)
- Mixed write/propose/chat semantics by command type (writes for log/review; propose for plan/shopping; chat for prep/swap)
- Library-anchor format `library:meals#{anchor}` (kebab-case of H2/H3 heading)
- `weekly_kcal_adjustments` additive list in progress.md frontmatter for `/weekly-review` adjustment history (Phase 3 D-22)
- Partner placeholder resolved to "Farva" in Phase 2 D-01

- Audit-discovered BLOCKER-01: meals.md and recipes.md restructured for executable anchor lookup
- Audit-discovered BLOCKER-02: macro-templates.md gained Session-Type Mapping table closing the CAL-02 chain
